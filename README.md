## 📋 Что это и зачем

Это закрытая панель для владельцев типографии «ПИРАНЬЯ». Здесь вы:
- Видите доходы и расходы за выбранный месяц
- Контролируете выплаты менеджерам
- Учитываете личные снятия (Дима, Игорь)
- Отслеживаете разовые траты по заказам
- Видите статистику по работе каждого сотрудника

**Версия:** 3.4.7 | **Скрипт:** `5_grist_closed_section.py`

---

## 🗂️ Как это работает (просто)

```
Вы выбираете месяц в №0 → все таблицы ниже показывают данные только за этот месяц

№0 Отчёты (якорь)
├─ №1 Категории расходов ← собирает суммы из:
│   ├─ №3 Транзакции (вручную/из API Эльбы - в планах)
│   ├─ №4 Зарплаты менеджеров
│   └─ ExtraCosts (разовые по заказам)
│
├─ №2 Контрагенты ← показывает, кому платили в выбранной категории (таблица скрыта за ненадобностью)
├─ №4 Зарплаты ← расчёт ЗП + бонусы + изъятия
└─ №5-№6 Справочники бонусов и изъятий
```

> 💡 Вы выбираете месяц в одном месте — ОТЧЁТЫ (МЕС.), все суммы пересчитываются автоматически.  
> 💡 Колонки для ручного ввода выделены жирным шрифтом (и нижняя таблица для ввода расходов) в UI интерфейсе, остальные колонки - авто.

## 📊 Таблицы и колонки

### 🧾 ExtraCosts — Разовые расходы по заказам
*Для учёта выплат фрилансерам, монтажникам, доставке*

**Колонки:**
- Кому — кто получил деньги
- Категория — Монтаж / Доставка / Замер / Дизайн / Расходники / Другое
- Стоимость — сумма выплаты
- Комментарий — подробности
- Заказ — к какому заказу относится (скрыто)
- Период — авто-подтягивается из даты заказа (скрыто)

> Эти расходы не попадают в общую выписку из Эльбы. Сумма автоматически добавляется в категорию «Разовые по заказам» в отчёте.

---

### 🏭 Productions — Справочник исполнителей
*База подрядчиков, фрилансеров, производств*

**Колонки:**
- Название, Код, Менеджер
- Телефон (авто-формат), Добавочный, Email, Telegram, Вконтакте, Доп. контакты и адрес
- Организация (поиск) — привязка к юрлицу из справочника организаций
- Деятельность, Заметки, Статус (Работает / Закрыт)

> 🔧 Телефон автоматически приводится к формату `7XXXXXXXXXX` при вводе.

---

### 📅 №0 MonthlyReports — Главный отчёт (вы выбираете месяц здесь)
*Точка входа: выбрали месяц → увидели все цифры за него*

**Колонки:**
- Начало Периода — 1-е число месяца (выбрать для нового месяца)
- Период — «Январь 2026» (удобное представление - авто)
- Конец периода — последнее число месяца (авто, скрыто)
- Статус — Открыт / Закрыт (выбор)
- План №1, План №2 — целевые показатели прибыли (ручной ввод)
- Бонус за план №1, Бонус за план №2 — 20 000 ₽ / 10 000 ₽
- Общий доход — сумма заказов за период
- Операционные расходы — сумма всех категорий расходов
- Чистая прибыль = Доход по заказам − Расход

> 🔧 Доход считается по-разному:
> - Открытый месяц: все заказы по дате создания (кроме «ОТБОЙ»)
> - Закрытый месяц: только выполненные заказы («ОК» / «ОК - отдать») по дате выдачи

---

### 🗂️ №1 ExpenseCategories — Категории расходов
*Список всех видов трат с итогами*

**Колонки:**
- Название категории — Офис / Налоги / Материалы / Маркетинг / Производство / Зарплатный фонд / Разовые по заказам
- Период — связь с отчётом (техническое, скрыто)
- Общая сумма — итог по категории

> 🔧 Как считается сумма:
> - «Зарплатный фонд» = сумма ЗП всех менеджеров (кроме владельцев)
> - «Разовые по заказам» = сумма из таблицы ExtraCosts
> - Остальные категории = сумма транзакций из Эльбы/вручную

---

### 👥 №2 ExpenseRecipients — Контрагенты (кому платили)
*Показывает, кому и сколько перевели в выбранной категории*

**Колонки:**
- Контрагент — название компании/ИП
- Период — связь с отчётом (техническое)
- Категория — связь с №1 (техническое)
- Сумма по контрагенту — сколько всего перевели
- ИНН, КПП, Тип (ИП/ООО) — подтягиваются из справочника

> Показывает только тех, кому были выплаты в выбранной категории и периоде, сейчас заполняется вручную

---

### 💸 №3 ExpenseTransactions — Детализация транзакций
*Все платежи: из Эльбы (позже по API) или вручную*

**Колонки:**
- Дата — дата операции
- Контрагент — кому/от кого
- Категория — на что потрачено
- Сумма = Списано (если есть) иначе 0
- Поступило, Списано — раздельно доходы/расходы
- Источник оплаты — ООО / ИП / Карта / Наличные
- Тип денег, Тип документа, № документа, Описание, Тип операции — детали из выписки
- УСН: доход / расход — для бухгалтера
- Реквизиты контрагента, Наши реквизиты — платёжные данные
- ID в Эльбе — защита от дублей
- Источник — Эльба / Банк / Ручной
- Период — подтягивается из даты (техническое)

> 🔧 Автоматика:
> - Сумма = значение из «Списано» (нам нужны только расходы)
> - Период = по первому дню месяца от даты транзакции
> - Категория «Производство» ставится авто, если контрагент является исполнителем

---

### 💰 №4 SalaryCalculations — Расчёт ЗП менеджеров
*Зарплата + премии + бонусы − свои заказы*

**Колонки:**
- Менеджер — сотрудник
- Период — месяц расчёта (авто)
- Оклад — из карточки сотрудника (авто)
- Премия исполнителю 10% — за заказы, где он был исполнителем (авто)
- Премия по марже 10% — только для Ивана, если маржа >30% (авто)
- Премия по марже 30% — только для Ивана, если маржа ≤30% (авто)
- Премия KPI — если доход ≥ плана (+20к / +10к) (авто)
- Разовые бонусы — сумма из таблицы бонусов (авто)
- Свои заказы — стоимость заказов, где клиентом указан "ПИРАНЬЯ" (есть в базе) (авто)
- Аванс — сколько уже выдано (ручной ввод)
- Итого ЗП — финальная сумма (авто)
- Остаток = Итого − Аванс (авто)

> 🔧 Для владельцев (роль ADMIN): Итого ЗП = 50% от чистой прибыли − изъятия (правый верхний угол)
> Визуализация зарплат сотрудников выведена в диаграмму в нижнем виджете раздела "ОТЧЁТ ЗП+" на левой панели.

---

### 🎁 №5 SalaryBonuses — Разовые бонусы
*Ручной ввод премий за особые задачи*

**Колонки:**
- Менеджер
- Дата — авто
- Сумма
- Комментарий
- Период — авто из даты (скрыто)
- Ссылка на расчёт — для связи (техническое, скрыто)

> 🔧 Чтобы отобразились все бонусы / штрафы менджера, нужно просто выбрать сотрудника в таблице "ЗП СОТРУДНИКАМ (ДЕТАЛИЗАЦИЯ)", тогда в правой части "БОНУСЫ / ШТРАФЫ" отобразятся все записи отфильтрованные по месяцу и менеджеру.

---

### 💼 №6 OwnerDrawings — Изъятия владельцев
*Учёт личных снятий средств Димы и Игоря за выбранный период*

**Колонки:**
- Владелец — Дмитрий и Игорь
- Дата — (авто)
- Сумма
- Назначение — «отпуск», «мебель» и т.п.
- Период — (авто из даты)
- Доля прибыли = 50% от чистой прибыли месяца (авто, скрыто)
- Остаток к выплате = Доля − все изъятия за период (авто, скрыто)

> 🔧 Самый правый верхний угол - мини-таблица, в которой авто рассчитываются все изъятия, доли, прибыль и остаток за период для Владельцев.

---

## 📈 Статистика по менеджерам (в №4)

Дополнительные авто-колонки в таблице зарплат:
- Средняя маржа % — средняя прибыльность заказов менеджера
- Средний чек — средняя стоимость заказа (целое число)
- Количество заказов — сколько заказов закрыл за период

> Учитываются только релевантные заказы (по статусу и периоду). Эта статистика выведена в виджет-диаграмму в правом нижнем углу раздела "ОТЧЁТ ЗП+" на левой панели.

## 🔐 Кто что видит

**Менеджеры:**
- ❌ Не видят этот раздел вообще
- Работают только с заказами, клиентами, прайсом

**Владельцы (Админы):**
- ✅ Видят все таблицы
- ✏️ Могут редактировать только поля ручного ввода:
  - В №3 (РАСХОДЫ (ТРАНЗАКЦИИ)): Дата, Контрагент, Категория, Сумма, Источник оплаты, Описание и др.
  - В №4 (ЗП СОТРУДНИКАМ (ДЕТАЛИЗАЦИЯ)): Оклад, Аванс
  - В №5-№6 (ИЗЪЯТИЯ ИЗ КАССЫ, БОНУСЫ / ШТРАФЫ): все поля, кроме авто-дат и формул

> 🔧 Все поля с пометкой «авто» или «формула» редактировать нельзя — они считаются системой. Редактирование формульных полей ограничено правами доступа.

## 🚀 Как начать работать

1. Откройте документ в Grist
2. В левой панели Раздел "ОТЧЁТ+"
3. В таблице **ОТЧЁТЫ (МЕС.)** выберите строку или создайте новую выбрав первое число месяца в колонке "Начало периода"
4. Все остальные таблицы автоматически покажут данные за этот месяц
5. Для ввода новых расходов: в **РАСХОДЫ (ТРАНЗАКЦИИ)** → добавьте строку
6. Для расчёта ЗП: данные подтянутся автоматически, проверьте ручные поля (оклад можно задать в справочнике "СОТРУДНИКИ", аванс тут же ручным вводом)

> 💡 Совет: наслаждайтесь автоматизацией

## ❓ Частые вопросы

О: Убедитесь, что в транзакциях (№3) у записей выбрана категория «Офис» и период совпадает с выбранным в №0.
