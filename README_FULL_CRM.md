# CRM-МОДУЛЬ: Автоматизированная система учёта заказов + справочники (Раздел для менеджеров)

Это полноценная CRM-система для управления заказами типографии, построенная на платформе Grist. Система включает в себя автоматизации, валидации, бизнес-логику и защиту от ошибок пользователей.

### Основные таблицы системы:

1. **Employees (Сотрудники)** - справочник сотрудников с ролями доступа
2. **Productions (Производства)** - справочник исполнителей (скрыт от пользователей)
3. **Organizations (Организации)** - справочник юридических лиц
4. **Products (Товары/услуги)** - каталог продукции с ценами и себестоимостью
5. **Clients (Клиенты)** - база клиентов с контактами и историей заказов
6. **Orders (Заказы)** - заказы с автоматическими вычислениями и валидацией
7. **OrderItems (Позиции заказа)** - отдельные позиции в заказах с автоматическим расчетом

### Автоматические вычисления и триггеры:

- **Автоматическое вычисление площади** в позициях заказа на основе ширины и высоты
- **Автоматический расчет суммы позиции** с учетом количества, цены за единицу и коэффициентов
- **Автоматический расчет себестоимости** позиции в зависимости от выбранного производства
- **Суммирование стоимости и себестоимости** по всем позициям в заказе
- **Автоматический расчет долга** по заказу (итого минус внесено)
- **Автоматическая генерация номера заказа** и даты создания при создании записи
- **Автоматическое проставление типа плательщика** (юр.лицо/физ.лицо) в зависимости от наличия организации у клиента
- **Автоматическое проставление приоритетного исполнителя** (производства) при выборе товара
- **Автоматическое проставление статуса и менеджера** в позициях заказа на основе данных заказа
- **Автоматическое обновление статуса заказа** на "Отдать в работу" при полной оплате

### Система валидации и контроля:

- **Проверка заполненности обязательных полей** в заказах (клиент и менеджер)
- **Проверка заполненности обязательных полей** в карточках клиентов (ФИО и хотя бы один контакт)
- **Проверка дубликатов клиентов** по телефону
- **Блокировка изменения статуса заказа** при незаполненных обязательных полях клиента
- **Блокировка ручного изменения вычисляемых полей** (суммы, себестоимости, площади и т.д.) для всех пользователей

### Умный поиск и фильтрация:

- **Умная строка поиска для организаций** на основе ИНН и краткого названия
- **Умная строка поиска для клиентов** на основе всех контактных данных (ФИО, телефон, email, telegram, VK, организация)
- **Автоматическая привязка организаций к клиентам** при импорте (малая часть, остальное руками в процессе поступления заказов)
- **Фильтрация заказов и позиций по клиентам, менеджерам, статусам, исполнителям, категориям**

### Защита данных и безопасность:

- **ACL-права доступа** ограничивают редактирование вычисляемых полей и некоторых других
- **Защита от удаления заказов** всеми пользователями
- **Блокировка изменения номера и даты заказа** после создания
- **Разграничение прав доступа** по ролям пользователей

### Технические особенности:

- **Proxy-колонки** обеспечивают надежное обновление триггерных формул
- **Технические колонки** скрывают сложную логику от пользователей
- **Условное форматирование** визуально подсвечивает важные данные и ошибки в заполненных данных
- **Цветовая индикация статусов и менеджеров** помогает быстро ориентироваться в заказах
- **Колонка метода расчёта** в позициях заказа показывает, как рассчитывается цена (за м², шт., пог.м.)
- **Система триггеров** обеспечивает мгновенное обновление связанных данных


# Инструкция по запуску скрипта автоматического создания базы данных Grist

## Подготовка

1. Убедитесь, что у вас установлен Python 3.7 или выше
2. Установите зависимости:

   source venv/bin/activate

   ```
   pip install -r requirements.txt
   ```

## Настройка

1. Убедитесь, что файл `.env` содержит корректные данные:
   - `API_KEY_GRIST`: ваш API ключ от Grist
   - `ID_DOC`: ID документа Grist, в котором будет создаваться база данных

## Запуск

Запустите скрипт командой:
```
python3 0_grist_db_setup_perp_full.py
```
Далее:  
0. Установить формулу для нормализации телефонных номеров

Далее запуск скриптов по порядку:
```
python3 1_grist_import_elba_orgs.py
python3 2_grist_import_clients.py
python3 3_grist_setup_price_structure.py
python3 4_grist_import_price.py
```

Далее:  
0. Выставить зависимые колонки для триггерных функций
1. Установить цветовое оформление для менеджеров
2. Установить правила для подсветки обязательных полей
3. Настроить права доступа
4. Настроить конфигурацию полей в карточках
5. Скрыть технические колонки

## Что делает скрипт

Скрипт автоматически создает в документе Grist:

1. Таблицу `Companies` (юридические лица) с полями:
   - Elba_ID, Name_Full, Name_Short, INN, KPP, Address_Legal, Address_Post, OKPO, OGRN, Contacts_JSON

2. Таблицу `Contacts` (контактные лица) с полями:
   - Full_Name, Phone, Email, Note, Default_Company (ссылка на Companies), Label_Search (формула поиска)

3. Таблицу `Orders` (заказы) с полями:
   - Order_Number, Date_Created (формула), Date_Deadline, Status, Manager (ссылка на Employees),
   - Contact (ссылка на Contacts), Payer_Type, Payment_Subject, Payment_Bill_Num, Delivery_Info,
   - Client_Note, Subject, Tags, Total_Sum, Total_Paid, Debt (формула), Cost_Total

4. Таблицу `OrderItems` (позиции заказа) с полями:
   - Order (ссылка на Orders), Description, Product (ссылка на Products), Contractor (ссылка на Companies),
   - Width, Height, Quantity, Area (формула), Unit_Price, Total_Price (формула), Unit_Cost, Total_Cost, Production_Note

5. Таблицу `Products` (прайс-лист) с полями:
   - Name, Base_Price, Calc_Method

6. Таблицу `Employees` (сотрудники) с полями:
   - Name, Short_Name

Также скрипт настраивает:
- Связи между таблицами
- Формулы для автоматических вычислений
- Права доступа (ограничение на удаление заказов) - настраиваем руками
- Представления (Views) для фильтрации заказов - настраиваем руками


## Список всех улучшений для стабилизации работы базы данных:

### 1. Защита от «пустоты» (NoneType Error)

Во всех математических формулах добавлена проверка `or 0`. Это предотвращает падение скрипта в ошибку, если менеджер еще не успел заполнить поле или случайно удалил значение.

* **Где:** `Debt`, `Area`, `Total_Price_Items`.
* **Результат:** Больше никаких красных треугольников с ошибкой `unsupported operand type`.

### 2. Автоматическое обнуление платежа

Добавлен триггер на колонку `Total_Paid`. Если менеджер полностью стирает сумму, система принудительно ставит `0` вместо пустоты.

* **Результат:** Корректный расчет долга сразу после создания заказа.

### 3. Гибкий ввод коэффициента (наценки)

Формула теперь принимает значение `Markup_Percent` как строку. Она сама удаляет знаки `%`, пробелы и меняет запятые на точки перед расчетом.

* **Результат:** Менеджер может писать «10», «10%» или «10,5» — система поймет всё.

### 4. Интеллектуальное округление (UP-to-10)

Внедрена функция `math.ceil()` для итоговой стоимости позиции. Сумма всегда округляется **вверх до ближайших 10 рублей**.

* **Результат:** Избавление от «копеечных» счетов (вместо 121 руб. будет 130 руб.).

### 5. Безопасный расчет площади

Формула площади обернута в `try-except`. Даже если в колонки ширины или высоты попадет некорректный текст, площадь просто станет равной `0`, не ломая расчет всей таблицы.

* **Результат:** Стабильная работа при частичном заполнении данных.

### 6. Умный авто-статус (Логика «Оплачено»)

В триггер статуса добавлена проверка типа данных `Debt`. Теперь система надежно переводит заказ в статус **«Отдать в работу»** только при выполнении двух условий:

1. Долг равен или меньше 0.
2. Сумма заказа больше 0 (чтобы не запускать в работу пустые строки).

---

**Итог:** База стала «стрессоустойчивой» к человеческим ошибкам и более удобной для бухгалтерии за счет округления. Все эти правки уже адаптированы под твой Python-скрипт.

### ⚡️ Обновления системы (версия 1.8.1) - КРАТКО

* **«Защита от дурака»**: Все формулы переписаны так, чтобы система не выдавала ошибок, если поля не заполнены или стерты.
* **Умный ввод чисел**: Поля (ширина, высота, наценка) теперь понимают и точку, и запятую. Наценку можно вводить со знаком `%`.
* **Авто-округление**: Стоимость позиции теперь автоматически округляется **вверх до 10 рублей** (удобно для расчетов и бухгалтерии).
* **Контроль оплат**:
* Если стереть сумму в «Оплачено», система сама поставит «0» (исключает ошибки в расчете долга).
* Статус **«Отдать в работу»** включается сам только при полной оплате и корректных данных.


* **Точность 2.0**: Все размеры (ширина/высота) принудительно хранятся в формате «0.00», что исключает путаницу в миллиметрах.

---

**Что это дает:** Скорость работы менеджера выше, ошибок в базе меньше, итоговые суммы в счетах выглядят аккуратнее.
