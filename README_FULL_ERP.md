# ERP-МОДУЛЬ: Финансовая панель (Закрытый раздел)

## 📋 Обзор системы

ERP-модуль — это админ-панель для владельцев типографии, предназначенная для:
- Управления операционными расходами
- Расчёта заработной платы менеджеров
- Контроля прибыли и убытков
- Учёта личных изъятий владельцев
- Сбора статистики продаж

## 🏗️ Архитектура взаимодействия
```
№0 (MonthlyReports) → период-якорь (итоговый отчёт по суммам из №1)
  ├─ №1 (ExpenseCategories) → категории для фильтрации (данные из №3 + №4 + ExtraCosts)
  │   └─ №2 (ExpenseRecipients) → контрагенты для фильтрации (данные из №3)
  │       └─ №3 (ExpenseTransactions) → точка входа данных из API или ручной ввод
  ├─ №4 (SalaryCalculations) → ЗП
  │   ├─ №5 (SalaryBonuses) → бонусы
  │   └─ №6 (OwnerDrawings) → изъятия
  └─ ExtraCosts → разовые расходы
```
### 1. Схема потока данных (Расчёт сумм)
*Откуда берутся цифры и как они считаются для отчёта*
```
┌─────────────────────────────────────────────────────────┐
│  №0 MonthlyReports (Итоговый отчёт)                     │
│  «Общий Расход» = SUM(№1.Общая сумма)                   │
└────────────────────┬────────────────────────────────────┘
                     ▲
                     │ (суммирует категории)
┌────────────────────┴────────────────────────────────────┐
│  №1 ExpenseCategories (Категории расходов)              │
│  «Общая сумма» = SUM(№3 + №4 + ExtraCosts)              │
└─────┬────────────────────┬──────────────────┬───────────┘
      │                    │                  │
      ▼                    ▼                  ▼
┌─────────────┐    ┌─────────────────┐  ┌──────────────┐
│ №3          │    │ №4              │  │ ExtraCosts   │
│ Expense     │    │ Salary          │  │ (Разовые по  │
│ Transactions│    │ Calculations    │  │  заказам)    │
│ (Транзакции)│    │ (ЗП менеджеров) │  │              │
└─────────────┘    └─────────────────┘  └──────────────┘
```

### 3. Итоговая схема (объединённая)

```
№0 (MonthlyReports) → период-якорь (итоговый отчёт по суммам из №1)
  │
  ├─ [ФИЛЬТРАЦИЯ] ───────────────────────────────────────┐
  │   №0 → №1 → №2 → №3 (иерархическая фильтрация)       │
  │   №0 → №4 ← №5 (ЗП и бонусы)                         │
  │   №0 → №4 ← №6 (ЗП и изъятия)                        │
  │   №0 → ExtraCosts (разовые расходы)                  │
  │                                                      │
  └─ [РАСЧЁТ ДАННЫХ] ────────────────────────────────────┘
      №3 + №4 + ExtraCosts → №1 → №0 (суммирование)
```

#### 🔧 Триггер: Нормализатор телефона (такой же как в CRM)

При вводе номера телефона система автоматически:
1. Удаляет все символы кроме цифр
2. Если номер начинается с `8` и содержит 11 цифр — заменяет `8` на `7`
3. Если номер начинается с `9` и содержит 10 цифр — добавляет `7` в начало
4. Возвращает номер в формате `7XXXXXXXXXX`

### №0. Таблица: **MonthlyReports** (Периодический отчёт)

**Назначение:** Основной периодический отчёт — точка привязки всех данных.

| Колонка | Тип | Описание |
|---------|-----|----------|
| Period_start | Date (Trigger) | Первый день месяца (2026-01-01) |
| Period | Text (Trigger) | «Январь 2026» — для выбора пользователем |
| Period_end | Date (Trigger) | Последний день месяца (2026-01-31) |
| Status | Choice | [Открыт, Закрыт] |
| Threshold_1 | Numeric | План №1 (прибыль) |
| Threshold_2 | Numeric | План №2 (прибыль) |
| Bonus_1 | Numeric | Бонус за план №1 (20000 ₽) |
| Bonus_2 | Numeric | Бонус за план №2 (10000 ₽) |
| Total_Income | Numeric (Formula) | **Доход** — сумма заказов за период |
| Operating_Expenses | Numeric (Formula) | **Расход** — SUM(№1.Total_By_Category) |
| Net_Profit | Numeric (Formula) | **Прибыль** = Доход − Расход |

#### 🔧 Автоматизации

**Period_start (авто-дата при создании):**
- При создании записи автоматически устанавливается первый день текущего месяца
- Время обнуляется (00:00:00)
- При ручном изменении сохраняется введённое значение

**Period (красивое название):**
- Автоматически генерирует название периода в формате «Месяц Год» (например, «January 2026»)
- Используется для выбора пользователем в интерфейсе

**Period_end (конец месяца):**
- Автоматически вычисляет последний день месяца на основе Period_start
- Например: для января 2026 → 31 января 2026

**Total_Income (Доход) — логика расчёта:**
- **Открытый месяц:** считаются все заказы по дате создания (`Date_Created`) за период, кроме статуса «ОТБОЙ»
- **Закрытый месяц:** считаются заказы по дате выдачи (`Date_Deadline`) за период, только со статусами «ОК» и «ОК - отдать»
- Суммируется стоимость всех подходящих заказов (`Total_Order_Amount`)

### №1. Таблица: **ExpenseCategories** (Категории расходов)

**Назначение:** Уровень 1 — категории операционных расходов.

| Колонка | Тип | Описание |
|---------|-----|----------|
| Name | Text | Категория (Офис, Налоги, Материалы, Маркетинг, Производство, Зарплатный фонд, Разовые по заказам) |
| Period | Ref: MonthlyReports | **Связь для формул** |
| Total_By_Category | Numeric (Formula) | Сумма по категории |
| Is_Template | Bool | Техническая (для фильтрации) |

#### 🔧 Автоматизации

**Total_By_Category — логика расчёта по категориям:**

1. **Зарплатный фонд:** суммирует `Total_Salary` всех менеджеров из SalaryCalculations за период, исключая владельцев (роль ADMIN)

2. **Разовые по заказам:** суммирует стоимость из ExtraCosts за период:
   - Для закрытого месяца: только заказы со статусами «ОК» / «ОК - отдать»
   - Для открытого месяца: все заказы

3. **Обычные категории (Офис, Налоги и др.):** суммирует транзакции из ExpenseTransactions по выбранной категории и периоду

### №4. Таблица: **SalaryCalculations** (Расчёт ЗП)

**Назначение:** Расчёт заработной платы менеджеров и исполнителей.

| Колонка | Тип | Описание |
|---------|-----|----------|
| Employee | Ref: Employees | Сотрудник |
| Period | Ref: MonthlyReports | Период |
| Base_Salary | Numeric (Trigger) | Оклад (из Employees) |
| Commission_Executor | Numeric (Formula) | Премия исполнителю 10% |
| Commission_Margin_10 | Numeric (Formula) | Премия по марже 10% (только старший менеджер) |
| Commission_Margin_30 | Numeric (Formula) | Премия по марже 30% (только старший менеджер) |
| KPI_Bonus | Numeric (Formula) | Премия KPI |
| One_Time_Bonuses | Numeric (Formula) | Выплаты / Штрафы |
| Own_Orders | Numeric (Formula) | Свои заказы |
| Drawings | Numeric | Аванс (ручной ввод) |
| Total_Salary | Numeric (Formula) | Итого ЗП |
| Remaining | Numeric (Formula) | Остаток ($Total_Salary - $Drawings) |

#### 🔧 Автоматизации расчёта ЗП

**Base_Salary (оклад):** автоматически копирует значение оклада из карточки сотрудника (Employees.Base_Salary)

**Commission_Executor (премия исполнителю 10%):**
- Находит все заказы периода, где сотрудник указан как исполнитель в позициях заказа (Production.Name = Employee.Name)
- Для закрытого месяца: только заказы со статусами «ОК» / «ОК - отдать» по дате выдачи
- Для открытого месяца: все заказы по дате создания, кроме «ОТБОЙ»
- Суммирует 10% от стоимости позиций заказа

**Commission_Margin_10 (премия по марже 10% — только старший менеджер):**
- Рассчитывается только для сотрудника с email `89045570154ivan@gmail.com`
- Находит заказы менеджера за период с маржой > 30%
- Суммирует 10% от стоимости этих заказов

**Commission_Margin_30 (премия по марже 30% — только старший менеджер):**
- Рассчитывается только для сотрудника с email `89045570154ivan@gmail.com`
- Находит заказы менеджера за период с маржой ≤ 30%
- Суммирует 30% от маржи этих заказов (Доход − Себестоимость)

**KPI_Bonus (премия KPI):**
- Не рассчитывается для старшего менеджера и владельцев (роль ADMIN)
- Если доход периода (Total_Income) >= План №1 → +20 000 ₽
- Если доход периода >= План №2 → +10 000 ₽
- Обе премии суммируются

**One_Time_Bonuses (бонусы):** суммирует все записи из SalaryBonuses для сотрудника за период

**Own_Orders (свои заказы):**
- Находит заказы, где в карточке клиента указана организация типографии
- Учитываются только заказы, где менеджером указан текущий сотрудник
- Суммирует стоимость этих заказов за период
- Для закрытого месяца: только статусы «ОК» / «ОК - отдать»

**Total_Salary (итого ЗП):**
- **Для менеджеров (EDITOR):** Оклад + все премии + бонусы − Свои заказы
- **Для владельцев (ADMIN):** 50% от чистой прибыли периода − сумма изъятий из OwnerDrawings
- **Для остальных:** Оклад + премии + бонусы − Свои заказы − Аванс

**Remaining (остаток):** Итого ЗП − Аванс (показывает сумму к выплате)

## 📈 Статистика менеджеров (SalaryCalculations)

### Дополнительные колонки статистики

| Колонка | Тип | Описание |
|---------|-----|----------|
| Avg_Margin_Percent | Numeric (Formula) | Средняя маржа в % |
| Avg_Check | Numeric (Formula) | Средний чек (int) |
| Order_Count | Numeric (Formula) | Количество заказов |

#### 🔧 Автоматизации статистики

**Avg_Margin_Percent (средняя маржа %):**
- Находит все заказы менеджера за период
- Для закрытого месяца: только статусы «ОК» / «ОК - отдать»
- Для открытого месяца: все заказы, кроме «ОТБОЙ»
- Вычисляет среднее значение маржи в процентах

**Avg_Check (средний чек):**
- Находит все заказы менеджера за период (с учётом статусов)
- Вычисляет среднюю стоимость заказа (округляется до целого)

**Order_Count (количество заказов):**
- Подсчитывает количество заказов менеджера за период
- Для закрытого месяца: только статусы «ОК» / «ОК - отдать»
- Для открытого месяца: все заказы, кроме «ОТБОЙ»
